# Database Design Document — Sprint 6.4: Portfolio Doctor Pro

This document specifies the database tables, properties, indexes, schema migrations, and security settings (RLS) required for Portfolio Doctor Pro.

---

## 1. Table Structures

### 1.1 `portfolio_snapshots`
Logs chronological audit scores for trend mapping.

| Column Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | PRIMARY KEY, DEFAULT gen_random_uuid() | Unique identifier |
| `user_id` | UUID | REFERENCES auth.users(id), NOT NULL | User account link |
| `health_score` | NUMERIC(5,2) | NOT NULL, CHECK (health_score >= 0 AND health_score <= 100) | Overall health rating |
| `risk_score` | NUMERIC(5,2) | NOT NULL | Raw risk index |
| `diversification_score` | NUMERIC(5,2) | NOT NULL | Diversification rating |
| `sector_score` | NUMERIC(5,2) | NOT NULL, DEFAULT 85.00 | Sector balance rating |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | Timestamp |

### 1.2 `portfolio_alerts`
Logs interactive, dismissible warnings generated by diagnostic audits.

| Column Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | PRIMARY KEY, DEFAULT gen_random_uuid() | Unique identifier |
| `user_id` | UUID | REFERENCES auth.users(id), NOT NULL | User account link |
| `alert_type` | VARCHAR(50) | NOT NULL | `IT_OVERWEIGHT`, `CASH_LOW`, `RISK_HIGH`, etc. |
| `message` | TEXT | NOT NULL | Warning message text |
| `status` | VARCHAR(20) | DEFAULT 'active', CHECK (status IN ('active', 'dismissed')) | Alert state |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | Timestamp |

### 1.3 `portfolio_recommendations`
Logs persistent AI recommendations with completion tracking.

| Column Name | Data Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `id` | UUID | PRIMARY KEY, DEFAULT gen_random_uuid() | Unique identifier |
| `user_id` | UUID | REFERENCES auth.users(id), NOT NULL | User account link |
| `recommendation` | TEXT | NOT NULL | Recommendation body text |
| `priority` | VARCHAR(20) | CHECK (priority IN ('high', 'medium', 'low')) | Importance rating |
| `status` | VARCHAR(20) | DEFAULT 'active', CHECK (status IN ('active', 'implemented', 'dismissed')) | Recommendation state |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | Timestamp |

## 2. Indexes
To ensure fast rendering of trend graphs and alert feeds, the following composite indexes are configured:
```sql
CREATE INDEX idx_portfolio_snapshots_user_date ON portfolio_snapshots(user_id, created_at DESC);
CREATE INDEX idx_portfolio_alerts_user_status ON portfolio_alerts(user_id, status);
CREATE INDEX idx_portfolio_recs_user_status ON portfolio_recommendations(user_id, status);
```

## 3. Row Level Security (RLS) Policies
Each table requires RLS configuration to isolate user portfolios:

### 3.1 snap_select / snap_insert
```sql
ALTER TABLE portfolio_snapshots ENABLE ROW LEVEL SECURITY;

CREATE POLICY select_own_snapshots ON portfolio_snapshots
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY insert_own_snapshots ON portfolio_snapshots
  FOR INSERT WITH CHECK (auth.uid() = user_id);
```

### 3.2 alert_select / alert_insert / alert_update
```sql
ALTER TABLE portfolio_alerts ENABLE ROW LEVEL SECURITY;

CREATE POLICY select_own_alerts ON portfolio_alerts
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY insert_own_alerts ON portfolio_alerts
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY update_own_alerts ON portfolio_alerts
  FOR UPDATE USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
```
